The provided sources outline the requirements and tasks for a **Home Weather Station** project, focusing on the **Problem Domain** from a **Black-Box perspective**. Below is a text-based breakdown of the information contained in the source material:

### **Project Overview**
The primary objective is to develop a **Raspberry Pi Weather Station**. While the system should be based on the official Raspberry Pi foundation's project guide, you are permitted to **enhance or modify** it by adding sensors or using different communication technologies, provided it meets the core purpose.

### **System Requirements**
*   **Hardware & Software:** The system must include the necessary hardware, embedded software, and **weather monitoring software** that runs on a user device such as a smartphone or laptop.
*   **Measures of Effectiveness (MoEs):** Specific performance metrics must include **communication range (Wi-Fi)** and **weather reading archiving**.

### **Methodology: MagicGrid Framework**
The project utilizes the **MagicGrid® framework** for Systems Engineering. 
*   **Black-Box Perspective:** This initial phase focuses on how the system interacts with its environment to solve challenges. It identifies **inputs, outputs, and functions** without concern for the internal structure or behavior.
*   **Problem Domain Pillars:** The analysis covers Stakeholder Needs (Requirements), System Context (Structure), Use Cases (Behavior), and Measures of Effectiveness (Parameters).

### **Assigned Tasks**
1.  **MBSE Planning:** Initial model-based systems engineering planning.
2.  **Stakeholder Needs:** Capture these directly in the model and create **5 to 10 specific requirements** for the station.
3.  **System Context:** Define the boundaries and external interactions of the system.
4.  **Use Cases:** Identify how the system is used and add **item flows** to the scenarios.
5.  **Measures of Effectiveness:** Define quantifiable characteristics of the system.
6.  **Review:** Conduct a final review of the model.

### **Required Outputs**
*   A project hosted in **Teamwork Cloud (TWC)**.
*   A **Table of Stakeholder Needs**.
*   **System Context Diagrams** (using BDD and IBD formats).
*   **Use Case Diagrams**.
*   **Scenarios** defined in Activity Diagrams (ACTs).
*   **MoEs** defined in Block Definition Diagrams (BDD).
*   A captured **Review** within the model.
